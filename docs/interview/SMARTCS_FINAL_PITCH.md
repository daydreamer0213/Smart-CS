# SmartCS 最终项目表达稿

## 一句话定位

SmartCS 是一个面向企业内部员工的多租户 HR 服务 Agent 后端工程样板：员工在 JWT 身份和文档受众边界内查询制度，获得来源引用；遇到例外先生成待确认草稿，确认后由 HR 完成可追踪的处理生命周期。

## 简历 bullet

- 独立构建 FastAPI 多租户 HR 服务 Agent，结合 JWT、角色化文档权限、文档导入、ChromaDB 与 BM25，提供带来源引用的制度检索问答。
- 实现“例外识别 -> 待确认草稿 -> 幂等确认建单 -> HR 指派/解决 -> 员工查询状态”闭环，以后端状态机约束 Agent，而非让模型直接写业务数据。
- 通过 pytest 覆盖认证、租户隔离、文档受众、检索降级与转人工生命周期，并提供可复现的本地实时演示脚本。

## 30 秒开场

> SmartCS 是我用来证明 Python AI 后端能力的 HR 服务 Agent 项目。员工先通过 JWT 进入自己的租户，Agent 只检索该员工可见的制度文档，并给出来源引用。制度没有覆盖的例外不会被模型编造，而是先形成待确认草稿；员工确认后，后端用幂等键创建正式请求，再由 HR 指派和解决。它展示的是 Agent 在企业身份、权限、RAG 证据和服务状态约束下如何真正落地。

## 诚实边界

SmartCS 不是已经上线的商业 HR SaaS，也不是完整 HRIS。它尚未接入 SSO/SCIM、真实 HRIS 或工单系统、通知 SLA、生产观测、CI/CD 和生产密钥治理。实时回答依赖已配置的 LLM、embedding 与制度文档；若模型返回 `503`，演示应失败并排查，不应把失败描述为成功。
