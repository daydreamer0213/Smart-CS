# SmartCS 项目路线图

## 里程碑 1：企业 HR 服务 Agent 基座 [状态：已完成]
**完成日期：** 2026-07-18
**目标：** 交付可演示、可测试的企业内部 HR 知识服务 Agent 后端，而不是包装成 FAQ 聊天机器人。

### 已交付能力

- [x] 文档导入：txt、md、pdf、docx、xlsx 解析、分块、嵌入和后台查看。
- [x] JWT 身份边界：用户注册、登录、刷新令牌、租户与角色后端校验。
- [x] 多租户 HR 知识服务：员工只能检索本租户且符合文档受众角色的制度来源。
- [x] 混合检索与来源治理：BM25 + Vector 检索、来源返回、后端规范化合法引用。
- [x] 受限 HR Agent：知识检索、澄清、待确认转人工草稿、员工工单状态查询。
- [x] 人工处理闭环：员工确认后创建正式请求，owner/admin 指派或解决，跨租户访问返回 403。
- [x] 工程交付：Alembic 迁移、pytest 回归、隔离数据演示脚本、运行手册与面试材料。

## 里程碑 2：文档智能与知识治理 [状态：已交付]
**目标：** 将基础文档导入升级为可评测、可治理的企业 HR 知识管道，使非标准制度文档经过解析、质量检查、版本管理和可追溯索引后再供 Agent 使用。

### M2-1：文档契约、合成语料与基准测试 [状态：已交付]
**交付日期：** 2026-07-18

- 已交付类型化解析契约、确定性的合成 HR 文档语料，以及离线可重复的 JSON 基准脚本。
- 当前基线覆盖 9 个语料：7 个解析成功、2 个错误、18 个必需事实中找到 16 个（88.89%）。这是合成语料上的当前基线，不是通用 PDF 准确率。
- 已记录扫描件、加密 PDF、混合文本/图像页、表格和双栏文档的已知差距，供后续阶段验证改进。

### M2-2：高级解析与 OCR 路由 [状态：已交付]
**交付日期：** 2026-07-19

- 保留 M2-1 的 9 语料、7 个解析成功、2 个错误和 16/18 事实基线，新增独立 structured benchmark，避免改进结果覆盖历史口径。
- 已交付 PyMuPDF 快速路径、Docling/Tesseract 高级路由、确定性结构化分块、页/章节/source lineage、质量门禁和 page-aware citation。
- 合成语料 gate 达到 18/18 parsed facts、18/18 chunk facts；表格关联、双栏顺序、PDF/DOCX/XLSX provenance 均通过，加密 PDF 被阻止索引并只返回安全原因。
- 结果仅证明合成语料验收通过；CPU OCR 仍存在运行差异。后续 M2-3/M2-4 已交付质量门禁和治理生命周期，Milestone 2 继续到 M2-5 检索评测完成。

### M2-3：解析质量门禁 [状态：已交付]
**交付日期：** 2026-07-19

- 解析结果按 `passed`、`review_required`、`failed` 分级，低质量文档不会静默进入员工知识库。
- 后台只返回受控诊断指标和警告，不泄漏解析器内部路径或异常细节。

### M2-4：文档治理生命周期 [状态：已交付]
**交付日期：** 2026-07-19

- 原件按内容寻址留存在 `D:\DevData\smartcs\documents`，文档家族、业务版本、索引代次和模型血缘可追溯。
- 新上传快照默认待审；批准后原子切换 `current_document_id`，员工 SQL 检索只放行当前、已批准、已生效、未过期且角色允许的版本。
- reindex 创建同业务版本的下一索引代次；解析、质量、Embedding、Chroma 或 BM25 失败时旧版本继续可用，部分外部写入执行尽力清理。
- 当前发布版本禁止删除；非当前版本删除时仅在没有共享引用后清理原件。

### M2-5：RAG 检索评测与交付 [状态：已交付]
**交付日期：** 2026-07-19

- M2-2 parser gate 的历史成功报告：9 个 fixture、8 个解析成功、1 个加密 PDF 被阻止索引，18/18 parsed facts、18/18 chunk facts 与 provenance 通过。CPU 上的 Docling/OCR 在当前内存压力下仍可能波动，因此该结果是固定语料与环境下的验收记录。
- M2-5 retrieval gate 与 parser gate 独立：8 个已索引 fixture、11 个仅由受信 facts 组成的 curated source chunks、12 条 golden queries、`top_k=3`。
- 实测 Recall@3 为 `11/12 = 91.67%`，MRR 为 `91.67%`，已召回来源 provenance accuracy 为 `100%`，门禁通过；`payroll-contact` 是唯一失败 query。
- 贡献统计为 BM25 11、vector 0。离线 HashEmbedding 非语义，只验证向量通路；评测不调用 FastGPT、LLM 或 LLM judge，不能据此声称混合语义检索质量。
- 报告和命令见 [M2-5 RAG 评测运行手册](../operations/rag-evaluation-m2-5.md)。这是 curated source-chunk 回归门禁，不是通用 PDF 准确率或生产 SLA。

### 交付范围

1. Docling/OCR：保留 PyMuPDF 数字 PDF 快速路径，为扫描件、复杂版式和表格增加高级解析路由。
2. 结构化元数据：保留页码、章节路径、表格标题与表头、解析器版本和索引代次。
3. 文档治理：增加原文件存储、文档版本、生效/失效日期、负责人、审核和发布状态。
4. 质量门禁：低质量、异常或解析不完整的文档进入人工复核，不静默发布到员工知识库。
5. 可量化评测：已建立 PDF/DOCX/XLSX 测试语料、解析基准和 RAG golden query 评测集。

### 验收门槛

- 每个已提交测试文档都有确定性的解析与分块断言。
- 扫描件、表格和复杂 PDF 的处理结果可查看、可诊断、可追溯。
- 所有可检索块保留来源页码和治理元数据，不破坏租户与受众权限。
- 检索 Recall@3 在约定测试集上达到至少 90%，完整回归测试通过；当前实测 91.67%，限制见 M2-5。

## 里程碑 3：真实 HR 工具接入 [状态：冻结待选]
**目标：** 证明 Agent 不只回答制度问题，还能在身份、权限和确认边界内调用真实 HR/OA 系统能力；不自研完整 HRIS。

### 计划范围

1. 建立供应商无关的 HR 工具适配器边界和标准错误模型。
2. 查询当前员工年假余额。
3. 查询员工本人 HR 工单，并复用 SmartCS 已有本地工单能力作为降级或演示路径。
4. 创建请假申请草稿；正式提交必须经过员工显式确认。
5. 查询组织关系、HRBP 或 HR 服务联系人。
6. 选择一个实际可调用的平台或沙箱完成端到端接入，其他供应商不做预留式实现。

### 验收门槛

- Agent 工具调用基于登录身份，不接受模型伪造员工 ID。
- 只读查询和写操作使用不同权限；写操作具备确认、幂等和审计。
- 外部系统超时、拒绝和不可用时有明确降级，不编造业务结果。
- 演示至少覆盖一次真实工具查询和一次确认后写操作。

## 里程碑 4：生产化加固 [状态：冻结待选]
**目标：** 为真实企业试点补足异步处理、可靠性、可观测、服务治理和企业身份能力。

### 计划范围

1. 后台异步导入：大文件上传与解析脱离请求生命周期，提供任务状态和取消边界。
2. 可靠处理：阶段级失败重试、幂等重新索引、死信/人工处理和上一可用索引回退。
3. 审计与可观测：端到端 tracing、业务指标、告警、审计留存和敏感字段脱敏。
4. 通知与 SLA：工单通知、超时升级、服务等级计时和管理员处理队列。
5. 企业身份：选择一个 SSO/OIDC 接入；SCIM 或组织同步按实际平台能力评估。
6. 工程交付：CI/CD、镜像发布、部署加固、备份恢复和生产密钥治理。

### 验收门槛

- 解析和索引任务可恢复、可重放、可观测，不因进程重启丢失状态。
- 关键 Agent、检索、外部工具和工单链路有 trace、指标和告警。
- 企业身份和数据权限在端到端测试中可验证。
- 部署、回滚、备份和密钥轮换有可执行运行手册。

### 条件性扩展

- 流式或双向通信：当前 HR 主入口使用普通 POST；仅在真实产品需要增量输出或双向实时事件时评估 SSE/WebSocket。
- Milvus：仅在 ChromaDB 的容量、并发或运维边界被实际压测证实后迁移。

## 范围之外

SmartCS 当前是企业 AI 应用工程样板，不宣称已上线为商业 HR SaaS，也不替代完整 HRIS。
